# 更新 Integration Log & 通知用戶

> **規則**：此工作流 **必須** 在任何 `notify_user` 調用 (回報進度或詢問用戶) 之前執行。

## 步驟

1.  **格式化對話紀錄**：
    - 準備對話日誌格式。
    - **標頭**：`> **[User]**: <用戶請求>`
    - **分隔線**：`> ---`
    - **回應**：`> **[角色名稱]**: <回應內容>`
    - **收尾**：`> ---`
    - **[關鍵禁令]**：嚴禁使用「省略」、「總結」或「細節請見完整對話」等字眼。必須忠實記錄技術決策、代碼清單與邏輯辯論。

2.  **更新 Integration.md**：
    - 使用 `multi_replace_file_content` 工具將新對話區塊 **追加** 至 `d:\AI\AI Discharge Care 360 Autogen\Integration.md` 末尾。
    - **關鍵原則**：每一條用戶指令與對應的專家回應都必須逐條記錄。

3.  **自審完整性 (Self-Audit)**：
    - 檢查是否有任何省略號 (...)。
    - 確保所有 `Coder` 產出的檔案清單與 `Critic` 的審核意見均已包含。

4.  **通知用戶**：
    - **僅在** 步驟 2 與 3 成功且符合「全量紀錄」原則後，才可執行 `notify_user`。
